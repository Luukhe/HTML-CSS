Listas Ordenadas

  1. Acordar
  2. Ligar para o Joao
  3. Tomar café
  4. Escovar os dentes
  5. Lavar roupa

Essas são listas ORDENADAS (ordered lists, tag: OL) que precisam ter os itens DENTRO da OL em forma de tag (tag: LI) que literalmente ORDENAM uma lista, ENUMERANDO-AS. Sendo assim, sempre que precisarmos adicionar um item na lista, a lista se REORDENARÁ e o item incluso se ALOCARÁ normalmente dentro da lista, fazendo com que não precisemos mudar item a item conforme a numeração.

Podemos também mudar o TIPO de "CONTADOR" da lista, usando o parâmetro TYPE e, também, dizer DE ONDE a lista deve COMEÇAR A CONTAGEM, usando o parâmetro START.

Listas NÃO Ordenadas

  • Pão
  • Manteiga
  • Arroz
  • Feijão

As listas NÃO ordenadas (tag: UL), é a lista onde não precisamos NOS PREOCUPAR com a ORDEM dos itens dentro da lista, onde não precisamos enumerar a sequência. Apenas DEMARCAR.

Embora não apareça como sugestão na UL, pode-se usar o parâmetro TYPE para mudar o formato dos índices da lista não ordenada, com os valores: DISC (que é o padrão), CIRCLE e SQUARE.

Lista de Definições

A definition list (tag: DL) funciona como se fosse um dicionário, onde a DL é a lista de definição, dentro dela teremos a tag DT (tag para o TERMO), e a tag DD (tag de DEFINIÇÃO do termo).

HTML (TERMO)

CSS
