Anotações sobre iframes

PRIMEIRAMENTE: o iframe é uma estrutura (que parece um quadro)INLINE (sim, se formos ver o tipo de display dele, será inline) que serve para CRIARMOS UMA "JANELA", COMO SE FOSSE UMA ABA NOVA que estará DENTRO da caixa do iframe. Para isso, é necessário colocarmos o "link" dentro do parâmetro "src" e, por padrão, o iframe virá também com o parâmetro 'frameborder="0"' (TUDO ESTÁ CONTIDO NA PÁGINA DE EXEMPLO iframe001).

NOTA: Existem alguns sites que BLOQUEIAM o uso de iframes por terceiros, como exemplificado no exemplo de página "iframe001", onde usamos o site do google para tentar colocar no iframe.

NOTA 2: Alguns navegadores de celular não possuem suporte ao iframe. Sendo assim, uma boa maneira de contornar isso, seria entre a abertura e o fechamento da tag, ou seja, DENTRO da tag MESMO (que nem nas aulas sobre vídeo), colocarmos um LINK para a página em questão.

SOBRE O FRAMEBORDER: O frameborder simplesmente só aceita dois valores, que são 0 e 1. O número "zero" seria para sem borda, e o número "um" com borda (qualquer outro valor acima de 1, será indiferente pois contará como 1).

(Podemos facilmente personalizar a borda com o parâmetro "border" sem problemas)

SOBRE TAMANHOS DE IFRAME: O tamanho padrão do iframe é de 300x150 (300 de largura e 150 de altura). Porém, logicamente, também conseguimos alterar a altura e largura dos iframes indo diretamente na tag e atribuindo um "height" e "width" que desejarmos. OU, também podemos usar css para alterarmos o tamanho, que nesse caso, o css terá prioridade sobre a propriedade inline.

SOBRE ROLAGEM NO IFRAME: Podemos também configurar a rolagem (caso haja) do iframe, usando o parâmetro "scrolling", que possui 3 valores:

  • AUTO: É o valor padrão do parâmetro
  • YES: Mesmo que o conteúdo esteja cabendo inteiro dentro do iframe, ainda assim apareceria barra de rolagem
  • NO: Exclui a rolagem do iframe mesmo que o conteúdo seja maior

CONTEÚDOS LOCAIS NO IFRAME: Para utilizarmos conteúdos próprios nos iframes, simplesmente colocamos o link da nossa página dentro do "src" (como foi utilizado no exemplo de página iframe002).

**** MUITÍSSIMO IMPORTANTE - COMO FAZER UM IFRAME NAVEGAR ENTRE DIVERSAS PÁGINAS (ex: iframe003): Para fazermos com que um iframe navegue (mostre) entre outras páginas ao clicarmos em determinados links na página, precisamos fazer com que o iframe tenha um parâmetro chamado "name", onde literalmente devemos nomeá-lo. Já os links precisarão usar o TARGET e, no valor do target, colocar o nome que foi atribuido no name do iframe!

***IMPORTANTE: Normalmente, mesmo as páginas já estando configuradas para serem mostradas no iframe quando clicadas no link, assim que a página for aberta (no caso, até se der um F5), por padrão, não será mostrado nada no iframe pois nada ainda terá sido selecionado, logo, assim que a página for carregada, será só um iframe vazio.

SENDO ASSIM, PODEMOS USAR AS ALTERNATIVAS:

  • usar o "src": pode-se simplesmente colocar algum link no "src" do iframe
  • *NOVO PARÂMETRO "srcdoc": Esse é um parâmetro bem peculiar, pois é um parâmetro que ACEITA LINHAS DE CÓDIGO HTML COMO VALOR!!!!!!!! (podendo assim, colocarmos h1, parágrafos e até img)

DICA IMPORTANTE (exemplo na página iframe004): Para deixarmos os iframes mais seguros em relação a ataques de outros sites, no que diz respeito à captação de dados coloca-se DENTRO DO IFRAME -sandbox="sandbox"-, sim, parâmetro sandbox com valor sandbox. Isso fará com que haja uma proteção maior sob outros sites para que eles não capturem dados de um formulário, por exemplo, facilmente!

OUTRO PARÂMETRO PARA AJUDAR: Um outro parâmetro (referrerpolicy="no-referrer") também serve para ajudar a proteger. Quando esse parâmetro é usado, quer dizer que os dados carregados dentro do iframe NÃO coletará nenhum tipo de informação do usuário para mecanismos de captura de dados de PROPAGANDA.

IMPORTANTE - CONFIGURANDO SANDBOX: Quando o parâmetro SANDBOX recebe o valor SANDBOX, ele bloqueará qualquer tipo de página para que não coletem os dados, incluindo páginas próprias, nossas mesmo, do próprio servidor.
Sendo assim, para que o iframe possa continuar protegido mas com "autorização" de coletar dados no iframe (levando em conta que a página no iframe seja do próprio servidor, localmente) é necessário usar os seguintes valores (dentro do parâmetro SANDBOX):

  • allow-same-origin: Permite conteúdos DE MESMA ORIGEM, ou seja, localmente.
  • allow-forms: Literalmente como o valor traduz, permite formulários
  • allow-scripts: Como o HTML5 executa scripts quando executa formulários (como foi no exemplo em questão, do iframe004), o valor permite executar formulários
