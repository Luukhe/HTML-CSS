ANOTAÇÕES SOBRE TABELAS EM HTML

SOBRE AS TABELAS: Primeiramente, criamos a tag "TABLE". A tag table irá delimitar a tabela. Todas as tabelas em HTML possuem a seguinte hierarquia:

        

            HIERARQUIA DE TABELAS (SIMPLES):

            TABLE = Tabela
                TABLE ROW = Linha de tabela
                    TABLE HEADER = Cabeçalho de tabela
                    TABLE DATA = Dado de tabela
            
            ATENÇÃO: Atentar-se à INDENTAÇÃO

    Que são representadas pelas tags:

  • TABLE: Representa a tag da tabela em geral
  • TR: Representa a table row (linha da tabela)
  • TH: Representa o TABLHE HEADER (cabeçalho da tabela)
  • TD: Representa o TABLE DATA (representa o dado da célula)

NOVA PROPRIEDADE: Na aula, o professor mostrou o "BORDER-COLLAPSE" (que por padrão, o valor é SEPARATE), que diz respeito ao "espacinho entre as bordas", e usou o valor COLLAPSE que fez com que as "células (os TD, no caso)" "se colapsarem", ficarem grudadas.

**ALINHANDO HORIZONTALMENTE O CONTEÚDO DOS "TDS": Para alinharmos horizontalmente qualquer dado/conteúdo dentro dos TDs, podemos simplesmente usar o TEXT-ALIGN (que por padrão já é LEFT, podendo usar também: CENTER e RIGHT).

***ALINHANDO VERTICALMENTE O CONTEÚTO/ITENS/DADOS DOS TDS: Usamos a propriedade "VERTICAL-ALIGN", que funciona muito bem para as células. O valor padrão da propriedade é MIDDLE, podendo usar também: top e bottom para alinhamentos na vertical.

TABELAS GRANDES: Sobre o uso de tabelas grandes, as tabelas também podem ter uma anatomia muito parecida com o HTML, no caso: possui também BODY, HEAD E FOOT.

        

            
                ANATOMIA PARA GABELAS GRANDES:

                        TABLE
                            THEAD
                                tr, td, th
                            TBODY
                                tr, td, th
                            TFOOT
                                tr, td, th

As tags são exatamente as tags citadas acima.

CAPTION: A tag CAPTION, é literalmente um TÍTULO da tabela, que ficará logo abaixo, sendo filho direto do table. O caption NÃO PODERÁ TER as demais tags (tr, td). Simplesmente escreve-se o título dentro da tag.

***IMPORTANTE; DEFININDO ESCOPO: Para designarmos um ESCOPO de um determinado dado (isto é, para dizermos que tal th será o título/escopo de uma determinada COLUNA de dados, como no exemplo de "População", "Estado" e "% da População", usados no exemplo tabela002) usamos a função (ou parâmetro, não sei) scope="col", lá no HTML.

Da mesma forma que podemos definir o escopo de uma coluna, também podemos definir o escopo de uma LINHA (exemplo: o "th" "TOTAL DE HABITANTES", cujo dado que representa o total de habitantes é na mesma LINHA e não COLUNA) usa-se scope="row", ao invés de COL.

CRIANDO EFEITO ZEBRADO EM TABELAS: O efeito zebrado é um meio utilizado para facilitar a nossa leitura em tabelas com muitos dados, pois nosso cérebro se confunde em situações como essa, onde nós nos perderíamos enquanto estivéssemos lendo. Para criar o efeito, utiliza-se a pseudo-classe "nth-child(2n)". O valor dentro do parêntesis corresponderá a QUANTAS LINHAS SERÃO PULADAS (exemplo usado na tabela002, onde foi usado nas "tr" do "tbody", juntamente com um background-color para diferenciar uma LINHA de outra). Pode-se também usar os valores "ODD" e "EVEN" para indicar APENAS as linhas de números ÍMPARES OU PARES (podemos até fazer configurações para odd e even simultaneamente, para usar cores diferentes ou várias outras configurações diferentes, também).

CABEÇALHO FIXO EM TABELAS: Para fazermos com que um cabeçalho fique FIXO em tabelas que tenham uma rolagem grande, a tag geral de tabela (TABLE) precisará ter um POSICIONAMENTO RELATIVO (position: relative), o posicionamento dos "th" do "thead" precisará ter um POSICIONAMENTO STICKY (position: sticky) e o mesmo também precisará ter um top: 0; (na verdade, coloca-se top: -1px; pois 0, fica ainda um pequeno pixel aparecendo atrás). Fazendo isso, os "th" em questão serão colados quando houver rolamento para baixo, MASSSS, como é dos "th" que estamos falando, os mesmos não possuem um background-color configurado. Logo, apenas as PALAVRAS serão levadas na rolagem. Sendo assim, configurando um background-color para os "th" (que seja da mesma cor que o thead, pois, senão, ele irá SOBREPOR o background do thead) fará com que possamos obter o efeito desejado.

**NOTA: Testei a mesma configuração acima (que foi usada nos "TH" do thead) diretamente no "THEAD" e funcionou normalmente. E, logicamente, não precisei configurar um outro background-color (para o th, no caso, como usado no exemplo), pois o thead já havia o seu próprio. Outra observação legal de mencionar é que o cabeçalho não rolará após o término da rolagem da tabela.
