SOBRE MEDIA QUERIES

DEFINIÇÃO GERAL: Media query, nada mais é do que uma estrutura do CSS que permite aplicar estilizações especificas para uma página web de acordo com certas condições, fazendo assim com esta página possa se adequar ao layout de tela em diferentes tamanhos e tipos de mídia. É sobre conseguirmos criar formas diferentes para apresentar o conteúdo em função do tamanho da tela!

SOBRE DIFERENTES LINKS DE CSS PARA A PÁGINA (MEDIA TYPES): Podemos ter diferentes links de CSS linkados à nossa página, onde cada link de CSS terá um estilo próprio para se adequar a uma determinada tela, ou seja, cada link servirá para um tipo de tela diferente, graças a um parâmetro chamado media, que contém diferentes valores para diferentes telas (conseguindo identificar, a partir dos valores de "media", para qual tela o link será direcionado). Temos então os seguintes valores para o media (QUE SÃO CHAMADOS DE MEDIA TYPES):

  • all: O valor all do parâmetro media, como o nome já sugere, fará com que o CSS seja direcionado a todos os tipos de tela. **Será a página de CSS que compartilhará configurações com todas as telas, no caso, seriam configurações que se repetiriam em medias diferentes.
  • screen: Como o nome sugere, fará com que a página de CSS seja direcionada apenas a telas (Genericamente qualquer tela).
  • print: Faz com que o css da página seja direcionado apenas a impressoras.
    **NOTA IMPORTANTE: Para impressoras, é aconselhável que alguns elementos da página sejam retirados (visualmente, usando display none), pois não fazem sentido estarem em uma página de papel impressa, tais como: menus, rodapés etc. Também é aconselhável usar algumas configurações inicialmente: Configuração global de margin e padding de 10px / font-size e line-height de 1.5em / font-family: Courier New.

*****SOBRE MEDIA FEATURES: Os principais media types vistos acima são o screen e print. Porém, é muito genérico falar apenas de "screen" pois existem muitos tamanhos diferentes de tela, e o media type screen engloba todas elas.
Logo, existe um complemento para os media types, que são as MEDIA FEATURES, que podemos traduzir como características de tela (telas diferentes), e são um complemento para media types, dentro dos media queries. Quando usamos media, podemos usar media types E TAMBÉM media features juntamente.
***PARA ADICIONARMOS OS MEDIA FEATURES, como dito acima, podemos usar types E features juntos. Para isso, adicionamos um "and" e, para orientações (vertical ou horizontal, por exemplo, um celular em pé ou deitado) deve-se especificar PORTRAIT ou LANDSCAPE, respectivamente.
**NOTA MUITÍSSIMO IMPORTANTE: Media types não possuem parêntesis, já os media features são indicados entre parêntesis isoladamente (caso tenhamos 3 media features, teremos que ter 3 parêntesis diferentes). Assim:

  • media="screen and (orientation: portrait)": Informará o media type (screen) e o media feature cuja orientação queremos configurar. No caso, é a configuração de um dispositivo que está EM PÉ, ou seja, orientation: portrait.
  • media="screen and (orientation: landscape)": Informará o media type (screen) e media feature cuja orientação queremos configurar. No caso, é a configuração de um dispositivo que está DEITADO, ou seja, orientation: landscape.

**PS: O MEDIA FEATURE É O ORIENTATION.

DICA: Após feitas as configurações gerais (configurações no media type "all") que servirão para todas as medias, podemos testar as configurações de orientation esticando ou alargando o navegador como se a dimensão fosse para telas deitadas ou telas em pé (para que possamos ver como ficará no tipo de media que queremos ver).

DICA 2, LEMBRAR: Sempre que for trabalhar com background-size contain e a imagem ficar bugada bem pequena, usar height 100vh e width 100vw.

***UNIFICANDO PÁGINAS DE CSS (Exemplo no exercício mq003): Às vezes podemos não querer criar tantas páginas de CSS diferentes (para telas diferentes) para apenas um projeto pequeno, tendo a possibilidade de unificarmos as páginas em apenas uma página de css, ou até mesmo em estilo local. Para isso, dentro da área de declarações de CSS, podemos simplesmente chamar o mesmo "media" do parâmetro do link, mas dessa vez com um "@" (que nem o charset, por exemplo) e especificar, então, o media type e o media feature, da mesma forma que fazemos quando chamamos um link. Então, dentro da área de declarações de CSS, ficaria assim:
@media screen and (orientation: portrait)
@media screen and (orientation: landscape)
@media all (até podemos fazer, mas seria apenas com o intuito de sinalizar, pois não faz diferença colocar ou não.)

Concluindo, temos 3 maneiras de fazermos nossas páginas de diferentes media queries:

  1. Criando 3 páginas de CSS diferentes para cada tela.
  2. Chamando o media de outras telas e fazendo tudo em estilo local.
  3. Chamando o media de outras telas dentro de uma só página de CSS (o que eu acho que é o que fazem normalmente).
