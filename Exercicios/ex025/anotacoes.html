SOBRE FORMULÁRIOS

Todo formulário tem que estar entre as tags FORM. Já um adendo: nem sempre funcionar é a melhor forma de fazer, o jeito que o primeiro exemplo será feito não está 100% bom, está funcional, mas não 100% bom (palavras do professor).

***SOBRE INPUTS: No exemplo (formulario01) foi usada a tag INPUT, dentro da tag form, dentro da tag parágrafo para APARECER AQUELA CAIXINHA DE DIGITAR INFORMAÇÃO e CAPTAR os dados de nome e sobrenome do exemplo. Existem MUITOS tipos de input para diferentes finalidades (pelo que eu entendi, normalmente os inputs vêm com a informação desejada), mas o que foi usado foi o INPUT TEXT (caixa de texto). Assim:

        
            
                <form>
                    <p>Nome: <input type="text" name="nome" id="nome"></p>
                    <p>Sobrenome: <input type="text" name="sobrenome" id="sobrenome"></p>
                </form>

**INPUT TEXT: Toda caixa de texto precisa ter NOME e ID. A diferença entre o "name" e o "id" nesse caso, será a identificação feita por cada linguagem. Em um caso, linguagens como PHP e HTML usam mais do "name" para a identificação do input, enquanto, em outro caso, linguagens como JavaScript o "ID" será mais utilizado (inclusive para o uso do label também). Logo, é extremamente necessário identificar todos os campos de entrada.

**INPUT SUBMIT: Como já sugere o nome, cria um botão de submit para o formulário. (da mesma forma que o input text, também foi colocado dentro de um parágrafo).
**É IMPORTANTE RESSALTAR QUE: quando a tag imput submit for usada, virá junto dela um parâmetro "VALUE", que é justamente onde devemos colocar o conteúdo escrito que queremos dentro do botão (EXEMPLO: Enviar).

DICA LEGAL PARA O FORM: Para desligarmos as sugestões de nomes que aparecem nos campos quando clicamos, usa-se o parâmetro "autocomplete=off" na tag form.

*****MUITÍSSIMO IMPORTANTE SOBRE O FORM: Os dados inseridos dentro de um formulário normalmente são mandados para alguma linguagem de programação, que irá variar de acordo com a linguagem usada para receber os dados do formulário (o professor no exemplo formulario01 usou a linguagem ou extensão .php). E para isso, usa-se um parâmetro que já vem por padrão no FORM chamado ** action="cadastro.js" **, sendo "registro.js" um SCRIPT a ser criado posteriormente, um programa para poder cadastrar os dados. E é assim que fazemos para enviar os dados de um formulário (action do form e botão de submit).

***IMPORTANTE; INCONVENIÊNCIAS: O navegador em si não sabe que o campo (a caixa onde digitamos a informação) está ligado aos nomes "Nome, Sobrenome" que colocamos e, é muito importante que que essa ligação exista para denominarmos o que cada campo de informação significa.

**PARA FAZERMOS ESSA LIGAÇÃO USAMOS A TAG "LABEL": Os labels, como o nome sugere, irão "etiquetar" os nomes que demos aos campos de informação e farão a ligação entre os nomes dados e o campo de dados (Exemplo: Nome e Sobrenome, do exercicio formulario01). Deve-se colocar as palavras "Nome" e "Sobrenome" DENTRO DA TAG LABEL e, a tag virá com o parâmetro "for" (para quem será o label), que é justamente onde devemos colocar o ***ID*** dado ao input, para que possamos fazer a ligação entre o nome e o campo.
NOTA IMPORTANTE: SEMPRE devemos usar label para fazer a relação entre qual é o objeto de formulário e qual é a etiqueta dele. Deve-se sempre relacionar um ao outro.

*SOBRE PARA ONDE VÃO OS DADOS ENVIADOS*: Na aula, o professor mostrou para onde são enviados os dados digitados nos campos (mesmo o script não funcionando). As informações estão contidas NO LINK!! Quando colocamos as informações nos campos e clicamos em enviar, analisando o link, pode-se notar que as informações colocadas nos campos foram parar ali.
Exemplo: ex025/registro.js?nome=Lucas&sobrenome=Vargas
Assim que clicamos no botão enviar, pode-se notar que o script (registro.js) é chamado, logo depois uma INTERROGAÇÃO, que servirá para dividir o endereço do arquivo que foi aberto, dos parâmetros extras que foram enviados que, no caso, foram enviados dois parâmetros: nome=Lucas&sobrenome=Vargas (esse "&" tem a mesma função de "E" normal mesmo: parâmetro "nome=Lucas", e parâmetro "sobrenome=Vargas"). Isso tudo é um padrão de envio de formulários.
NOTA - MÉTODOS DE ENVIO ATRAVÉS DE FORMULÁRIOS: Existem dois métodos que são métodos simples para envios de formulário (o método é um parâmetro "method" que pode ser colocado no FORM):

  • method="GET": Esse é o primeiro método e é o método padrão, que é o qual já estamos utilizando (o método que aparece no link os dados colodados nos campos). NOTA: É utilizado quando os dados NÃO forem sensíveis. Só deve ser usado até 3 mil bytes. Não deve ser usado para envios de FOTOS ou qualquer outro tipo de arquivo.

  • method="POST": Esse método irá esconder os parâmetros nos links. Ou seja, os dados colocados nos campos do formulário não aparecerão no link. PS: ISSO NÃO TORNA NADA MAIS SEGURO. NOTA: É utilizado quando forem DADOS SENSÍVEIS. Utilizado para dados de mais de 3 mil bytes. Utilizado para ENVIO DE ARQUIVOS.

**IMPORTANTE - SOBRE CAMPOS VAZIOS: Normalmente, se os campos de dados de um formulário estiverem vazios e a pessoa clicar em enviar, os dados SERÃO enviados, e isso não pode acontecer, não é algo interessante.
Para que isso não aconteça, precisamos fazer com que os campos sejam obrigatórios. Fazemos isso simplesmente adicionando um parâmetro DENTRO DO INPUT DO DADO, parâmetro: required.

**IMPORTANTE - SOBRE CARACTÉRES MINIMOS OU MÁXIMOS: Já nos deparamos com diversos sites em que no cadastro do site, era requerido um número mínimo de caractéres, e/ou que havia um número máximo de caractéres, seja para usuário ou senha. Isso pode ser feito usando DOIS PARÂMETROS:

  • minlenght: Parâmetro para ser usado DENTRO DO INPUT DO DADO DESEJADO para determinar o mínimo de caractéres que o dado precisa ter.
  • maxlenght: Parâmetro para ser usado DENTRO DO INPUT DO DADO DESEJADO para determinar o máximo de caractéres que o dado precisa ter.

    • Também é interessante saber os seguintes parâmetros:

  • size="": Pode-se também definir o tamanho da caixa onde colocamos o dado. O valor colocado será o valor do tamanho de caractéres (exemplo: um "size com valor 10, a caixa terá um tamanho de 10 caractéres").
    PS: não é porque o valor de uma caixa é de 10 caractéres que ele não aceitará passar de 10, o valor do size será apenas o tamanho. Passando o valor do size, será criado uma rolagem.
  • placeholder:="" Este é um parâmetro que coloca aquela dica do que é necessário escrever no campo, que some quando começamos a escrever (Exemplo: no campo de usuário quando está em branco, uma dica estará escrita dizendo "nome de usuário", que deverá ser escrito dentro do valor do parâmetro placeholder).

**DICA MUITO IMPORTANTE - AUTOCOMPLETE NOS INPUTS: O parâmetro autocomplete usado no form que serve para ligar ou desligar a sugestão de usuarios já usados/digitados, também pode ser usado nos inputs e possuem MUITO mais valores do que apenas os "on" e "off" do form. Nos inputs, pode-se sugerir coisas bem mais específicas que ficaram guardados no navegador, pois o navegador guarda as informações digitadas em diferentes "categorias", podendo assim usar o autocomplete apenas para sugerir coisas específicas já usadas como senhas, número de telefone entre outros dados.

****TIPOS DE INPUT USADOS ATÉ ENTÃO****:

  • INPUT TEXT: Como o nome sugere, usado para textos do tipo usuarios, nicks, nome, sobrenome etc;
  • INPUT PASSWORD: Como o nome sugere, usado para SENHAS. O texto digitado no input password fica automaticamente oculto (tendo a opçãozinha do olho para mostrar ou ocultar).
  • INPUT SUBMIT: Como o nome sugere, cria um botão de enviar, que enviará as informações contidas nos outros inputs.
  • INPUT RESET: Como o nome sugere, cria um botão que irá limpar o que foi digitado nos campos dos inputs.
  • INPUT NUMBER: Como o nome sugere, será um input específico para receber números.
    NOTA: Input text aceita letras e números. Mas, para um campo onde apenas números devem ser colocados, vem ao caso usar justamente o input number.
    **PARÂMETRO "STEP" PARA O INPUT NUMBER: Sem esse parâmetro, números "quebrados" não são aceitos. Esse parâmetro dita de quanto em quanto o número poderá ser selecionado (Exemplo: step com o valor 0.25, os valores poderão ser escolhidos de 0.25 em 0.25, tipo 5.25, 5.5, 5.75 etc)
    NOTA: Com esse valor atribuido, apenas valores de 0.25 em 0.25 serão válidos. Um valor de "5.8", por exemplo, não será válido pois não está no intervalo determinado.
    **PARÂMETRO "VALUE" PARA O INPUT NUMBER: Esse parâmetro faz com que o valor colocado nele mesmo, seja mostrado inicialmente no campo (ISSO SOBREPÕE PLACEHOLDER - O VALUE TAMBÉM FUNCIONA PARA DATAS).
  • INPUT MONTH: Como o nome sugere, é voltado para escolher o uso de meses.
  • INPUT DATE: Como o nome sugere, cria um input para uma data completa, com direito a um calendarinho pra escolher (dia/mês/ano).
  • INPUT TIME: Permite escolher a hora desejada pelo usuário.
  • **INPUT DATETIME-LOCAL: Esse input não foi mostrado na aula com o professor, fui eu mesmo procurando e fuçando que encontrei e achei interessante.Esse input permitirá que o usuário coloque data e hora JUNTOS no mesmo input.
