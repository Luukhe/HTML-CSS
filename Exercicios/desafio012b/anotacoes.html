PÁGINA VOLTADA PARA MINHAS ANOTAÇÕES SOBRE O PROJETO

  • TEXT-TRANSFORM: UPPERCASE; Pode-se transformar o texto todo em maiúsculo, assim como também em minúsculo ou capitalizar as primeiras letras.
  • FONT-VARIANT: SMALL-CAPS; Faz com que as letras também fiquem maiúsculas, MAS as PRIMEIRAS LETRAS de cada palavra ficam MAIORES (Foi o que o professor usou no h1 do desafio em questão, o qual eu fiz a gambiarra de colocar as iniciais das palavras em span e, no CSS, aumentar o tamanho das letras).
  • PADDING USANDO VH COMO MEDIDA: Na aula foi utilizado o padding usando vh como medida. Vale anotar pois é interessante, já que o vh usa sempre o tamanho atual da tela, e em dispositivos menores (ou bem maiores), o espaçamento que o padding faria seria proporcional às diferentes telas.
  • ***IMPORTANTE: FONT-SIZE USANDO "VW" COMO MEDIDA: Na aula, o professor utilizou o font-size para o H1 e as sections usando "VW" como medida, que de forma parecida com o VH (view height), o VW quer dizer VIEW WIDTH. Logo, ao invés de usarmos a medida EM como é de costume, podemos usar a medida BASEADA NA LARGURA DA TELA, fazendo assim com que a fonte mude de tamanho proporcionalmente à tela. (O professor também enfatizou que não é a melhor maneira, mas ainda assim, é um recurso.)

Só relembrando: display inline-block será uma caixa que se adapta ao tamanho do conteúdo!
